# v1.1 Cloud Sync

v1.1 starts the online foundation without changing the live v1.0 App Store build.

## Scope

- Keep the game playable as guest.
- Send completed game summaries to Supabase only when the user has a session.
- Queue authenticated game results locally if the network or server call fails.
- Keep server-owned totals in Postgres through `submit_game_result(p_result jsonb)`.
- Restore account totals, streak days, weekly progress, and achievements from Supabase at sign-in.
- Keep guest progress and each signed-in user's local snapshot in separate storage keys.
- Let signed-in users permanently delete their account and cloud data in the app.
- Do not push this branch to GitHub until v1.0 App Review/release is settled.

## Client Payload

The app sends one summary at the end of a completed game:

- `client_result_id`
- `mode`
- `difficulty`
- `score`
- `awarded_score`
- `targets_solved`
- `target_count`
- `operation_count`
- `duration_seconds`
- `hint_used_count`
- `puzzle_key`
- `completed`
- `par`
- `board_size`
- `source_numbers`
- `target_values`
- `locale`
- `client_finished_at`
- `client_utc_offset_minutes`

The client does not directly update totals, streaks, weekly scores, or achievements.

## Required Environment

Create a local `.env` file from `.env.example`:

```text
EXPO_PUBLIC_SUPABASE_URL=https://xuggavmtqgsxazbccfxj.supabase.co
EXPO_PUBLIC_SUPABASE_PUBLISHABLE_KEY=...
EXPO_PUBLIC_AUTH_REDIRECT_URL=islem://auth/callback
EXPO_PUBLIC_TURNSTILE_SITE_KEY=...
EXPO_PUBLIC_TURNSTILE_URL=https://aerkut-ops.github.io/islem-mobile/turnstile.html
```

The Turnstile site key is public and is included in the app build. The Turnstile secret belongs only in Supabase Auth settings. Never commit `.env`, Turnstile secrets, service role keys, database passwords, Apple keys, or EAS secrets.

Add `islem://**` to Supabase Auth redirect URLs before testing email magic links.
For the separately installed local device build, use `islem-dev://auth/callback` and add `islem-dev://**` to the same allowlist.

After applying the migration, verify that the `cleanup-expired-score-events` job is active under Integrations > Cron > Jobs. It runs daily at 03:17 UTC and removes only raw game events whose 90-day retention period has ended. Permanent player statistics, daily progress, weekly/monthly summaries, and achievement unlocks remain intact.

## Account Deletion

Deploy the authenticated Edge Function after linking the project:

```text
npx supabase functions deploy delete-account --project-ref xuggavmtqgsxazbccfxj --use-api
```

The function requires a valid user JWT and an explicit confirmation value. It deletes the authenticated `auth.users` row through the server-side admin client; foreign-key cascades remove the related profile, scores, streak, achievements, and game summaries. Never expose a Supabase secret key in the app.

## Email Delivery

Custom SMTP is enabled for the v1.1 beta account flow. A magic link requested from the installed iOS app was delivered through Gmail SMTP, opened the `islem-dev://auth/callback` route, signed in the development test account, and remained signed in after the app was fully closed and reopened.

The Gmail sender is suitable for limited beta testing only. Replace it with a transactional email provider and a verified sending domain before increasing authentication email limits or onboarding users at scale. Keep SMTP passwords and provider API keys outside the repository.

The bilingual magic-link email source is versioned at `supabase/templates/magic-link.html`. Keep the hosted Supabase Auth subject and template synchronized with this file when the copy or design changes.

## Bot and Abuse Protection

Email-link and password sign-in can be protected with Cloudflare Turnstile. The native app opens the HTTPS challenge page at `store-web/turnstile.html` in a constrained WebView, receives a single-use token, and passes that token to Supabase Auth with the sign-in request.

Before enabling CAPTCHA in production:

1. Create a managed Turnstile widget in the Cloudflare dashboard.
2. Restrict the widget hostname to `aerkut-ops.github.io`.
3. Put the public site key in the EAS production environment.
4. Select Turnstile under Supabase Authentication > Bot and Abuse Protection and enter the secret key there.
5. Build and test the app before enabling CAPTCHA for all users.

Use Cloudflare's official test keys only during development. Never ship a production build with a test key, and never put the secret key in the app or repository.

When the official always-pass Turnstile site key is present in a development build, the app sends Cloudflare's official dummy token directly to Supabase instead of opening the WebView. This keeps local account testing deterministic while Supabase uses the matching test secret. Release builds never use this shortcut; a real site key opens the hosted WebView challenge and still fails closed after a 20-second timeout.

## Native Build Note

The Turnstile screen uses `react-native-webview` 16, which matches React Native 0.86's new architecture. This version is intentionally excluded from Expo's recommended-version validation in `package.json`.

On iOS, `expo-build-properties` sets `buildReactNativeFromSource` to `true`. The setting avoids native linker incompatibilities between WebView 16 and Expo's prebuilt React Native core. Clean local and simulator builds were verified with this configuration. iOS builds take longer because React Native is compiled from source; do not remove the setting without repeating a clean native build and the complete authentication test.

## Security Verification

Run the read-only Row Level Security check before a TestFlight build:

```text
npm run check:cloud
```

The command signs in with both Keychain-backed test accounts and verifies that profiles, player statistics, daily progress, raw score events, achievements, and deletion requests are isolated per user. It also verifies that anonymous requests cannot read those tables. Weekly and monthly score tables are intentionally excluded from private-row assertions because authenticated players use them as shared leaderboards.

When the local environment uses Cloudflare's official always-pass test key, this check sends the matching official dummy token to Supabase. It does not bypass a real production widget: production Turnstile tokens are single-use and cannot be stored in the repository.
